LibreTranslate API

Open-Source Übersetzungsserver mit REST API. Kann selbst gehostet oder als öffentliche Instanz genutzt werden.

Server-URL
Basis-URL ohne Pfad, z.B. https://translate.max oder http://localhost:5000. Der Endpunkt /translate wird automatisch angehängt.
Sprachen
30+ Sprachen, automatische Erkennung
API-Key
Nur nötig wenn der Server Authentifizierung erfordert
Vorteil
Schnell, keine GPU nötig, ideal für Standardübersetzungen
API-Referenz
POST /translate
Request
{
  "q": "Hello world",
  "source": "auto",
  "target": "de",
  "format": "text",
  "api_key": ""
}
Response
{
  "translatedText": "Hallo Welt",
  "detectedLanguage": {
    "confidence": 98,
    "language": "en"
  }
}
Fehler
HTTP 403: Ungültiger API-Key
HTTP 429: Rate-Limit erreicht
HTTP 500: Server-Fehler

LM Studio API

Lokales LLM mit OpenAI-kompatibler API. Ideal für kontextbezogene Fachübersetzungen mit voller Datenkontrolle.

Server-URL
Basis-URL des LM Studio Servers, z.B. http://192.168.178.45:1234 oder http://localhost:1234. Der Pfad /v1/chat/completions wird automatisch angehängt.
Modell
Muss in LM Studio geladen sein. Klicke auf den Aktualisieren-Button um verfügbare Modelle abzurufen.
Empfohlene Modelle
Qwen 3 liefert exzellente Übersetzungsqualität mit zuverlässigem JSON-Output. Empfohlen: qwen3-next-80b-a3b-instruct-mlx (Apple Silicon) oder andere Qwen 3 Instruct-Varianten ab 12B+. Alternativ: Gemma 3 12B.
Temperatur
0.1 = wörtlich und konsistent (Standard). 0.3 = natürlicher Stil. 0.5+ = kreativ, aber weniger vorhersagbar. Für Fachübersetzungen unter 0.3 bleiben.
Max Tokens
2000 reicht für die meisten Texte. Bei langen Absätzen auf 4000 erhöhen. Das Modell muss mindestens so viel Kontext unterstützen.
CORS
In LM Studio unter Einstellungen "Enable CORS" aktivieren, sonst blockiert der Browser die Anfragen.
Fachkontexte
Kfz, Technik/IT, Medizin, Recht -- wählbar im Side Panel. Beeinflusst die Terminologie der Übersetzung.
System-Prompts je Fachkontext

{source} und {target} werden durch die gewählten Sprachen ersetzt.

Allgemein
Du bist ein präziser Übersetzer.
Übersetze den Text von {source} nach {target}.
Natürliche, flüssige Übersetzung. Formatierung beibehalten.
Antwort als JSON: {"translation": "...", "alternatives": [...]}
Kfz / Automotive
Kfz-Fachübersetzer {source} nach {target}.
- NIEMALS übersetzen: Teilenummern, OE-Nummern, Codes, Markennamen
- Korrekte Fachbegriffe: Control arm = Querlenker,
  Tie rod end = Spurstangenkopf, Ball joint = Traggelenk,
  Wheel bearing = Radlager, Brake caliper = Bremssattel
Antwort als JSON mit optionalem context_notes Feld
Technisch / IT
Technischer Fachübersetzer {source} nach {target}.
- Absolute technische Präzision
- Englische Fachbegriffe belassen (API, Cache, Backend, etc.)
- Code und Variablennamen NIEMALS übersetzen
Antwort als JSON: {"translation": "...", "alternatives": [...]}
Medizin
Medizinischer Fachübersetzer {source} nach {target}.
- Exakte medizinische Terminologie
- Lateinische/griechische Fachbegriffe beibehalten
- Höchste Präzision bei Dosierungen und Maßeinheiten
Antwort als JSON mit optionalem context_notes Feld
Recht / Juristisch
Juristischer Fachübersetzer {source} nach {target}.
- Exakte juristische Terminologie des Zielrechtssystems
- Länderspezifische Rechtsbegriffe (deutsches Recht)
- Im Zweifel: wörtliche Übersetzung mit Anmerkung
Antwort als JSON mit optionalem context_notes Feld

SWT Cache Server

Zentraler Übersetzungs-Cache auf Basis von Python/FastAPI. Speichert alle Übersetzungen serverseitig und macht sie geräteübergreifend verfügbar.

Funktionsweise
Übersetzungen werden per SHA-256-Hash der Domain zugeordnet und als Dateien gespeichert. Kein separater Index nötig
Cache-Modus
Bestimmt die Reihenfolge der Abfrage: Server/Lokal/Beides. Bei "Nur Server" wird kein LocalStorage verwendet
Auto-Laden
Wendet beim Seitenaufruf automatisch vorhandene Cache-Übersetzungen an, ohne manuellen Klick
Timeout
Maximale Wartezeit für Server-Antworten. Bei Überschreitung wird der Fallback-Modus genutzt
API-Referenz
POST /cache/bulk -- Speichern
{
  "url_hash": "a1b2c3d4e5f6",
  "items": {
    "hash1": ["original_b64", "translated_b64"],
    "hash2": ["...", "..."]
  }
}
POST /cache/get -- Abrufen
{
  "url_hash": "a1b2c3d4e5f6",
  "hashes": ["hash1", "hash2"]
}
Weitere Endpunkte
GET /cache/url/{hash} Stats pro Domain
GET /cache/url/{hash}/all Alle Einträge
DEL /cache/url/{hash} Domain löschen
GET /cache/urls Alle Domains
GET /stats Gesamtstatistik
GET /health Health-Check
Hash-Berechnung
url_hash (12 Zeichen, Ordnername):
  SHA256(hostname ohne www)[:12]
  z.B. "svelte.dev" → "a1b2c3d4e5f6"
  "www.github.com" → sha256("github.com")[:12]
  Nicht-Standard-Port bleibt: "localhost:3000"

trans_hash (64 Zeichen, Dateiname):
  SHA256(origin + pathname + text + ":langPair")
  z.B. SHA256("https://svelte.dev/docs"
    + "Hello world" + ":en:de")

Fallback bei HTTP: djb2-Hash (kein crypto.subtle)
