1. What is Smart Translator?

Smart Translator is a browser extension that translates web pages and selected text directly in the browser. All data stays local -- no external cloud services are used.

Full Pages Translate entire web pages with a single click
Text Selection Instantly translate selected text
Side Panel Translation panel alongside the web page
Cache Store and reuse translations centrally

2. Translation Backends

Smart Translator supports two different translation backends. You can switch between them in the settings.

LibreTranslate

  • Open-source translation engine
  • Fast, no GPU required
  • 30+ languages
  • Automatic language detection
  • Ideal for general texts
  • Self-hosted (Docker) or public

LM Studio

  • Local AI model (LLM)
  • GPU recommended (NVIDIA/Apple Silicon)
  • Domain contexts: Automotive, Medical, Legal, IT
  • Alternative translation suggestions
  • Ideal for specialized translations
  • Full data control, no cloud

3. Setting up LibreTranslate

Option A: Docker (recommended)

The easiest way to run LibreTranslate is via Docker:

docker run -d -p 5000:5000 libretranslate/libretranslate

Once started, the server is available at http://localhost:5000.

Option B: Public Instance

You can also use a public instance. Be aware of possible rate limits and privacy policies. A list can be found on the LibreTranslate GitHub page.

Settings

Enter the server URL in the Settings:

Server URL http://localhost:5000/translate
API Key (leave empty if not required)

How the API Works

Smart Translator sends translation requests as JSON to the server:

Request to the Server
POST /translate
Content-Type: application/json

{
  "q": "The quick brown fox",
  "source": "en",
  "target": "de",
  "format": "text"
}
Response from the Server
HTTP 200 OK

{
  "translatedText": "Der schnelle braune Fuchs",
  "detectedLanguage": {
    "confidence": 97.5,
    "language": "en"
  }
}

4. Usage

Select and Translate Text

Select any text on a web page. A small translation icon will appear. Click on it to instantly translate the selection. The result is displayed in a tooltip.

Translate Full Page

Click the extension icon in the toolbar, then click "Translate Page". All visible text will be translated in batches. Progress and speed depend on your backend.

Side Panel

The Side Panel opens alongside the web page and offers:

  • Text input with instant translation
  • Page translation with progress indicator
  • Translation history
  • Cache management with statistics
  • Domain context selection (with LM Studio)

Popup

The popup (click on the extension icon) provides quick access to:

  • Enter text and translate directly
  • Translate page / restore original
  • Open Side Panel
Part 2 -- Best Results with LM Studio

5. Setting up LM Studio

Step 1: Install LM Studio

Download and install LM Studio. Then download a language model -- recommended for translations:

  • Qwen3-Next 80B-A3B (MLX) -- best quality on Apple Silicon, MoE model (only 3B active parameters)
  • Gemma 3 12B -- good balance of quality and speed
  • Qwen 2.5 14B -- strong with multilingual texts
  • Llama 3.1 8B -- fast, for older hardware

Step 2: Start the Server

Start the local server in LM Studio (tab "Local Server"). By default it runs on port 1234.

Step 3: Settings

Server URL http://localhost:1234
Model (loaded automatically)
Temperature 0.1 -- low = precise
Network Tip: If LM Studio is running on another machine in the network, use its IP address, e.g. http://192.168.178.45:1234

How the API Works

Smart Translator uses the OpenAI-compatible Chat API of LM Studio:

Request to LM Studio
POST /v1/chat/completions
Content-Type: application/json

{
  "model": "gemma-3-12b-it",
  "messages": [
    {
      "role": "system",
      "content": "Translate EN to DE.
        Reply as JSON."
    },
    {
      "role": "user",
      "content": "The quick brown fox"
    }
  ],
  "temperature": 0.1,
  "max_tokens": 2000
}
Response from LM Studio
HTTP 200 OK

{
  "choices": [{
    "message": {
      "content": "{
        \"translation\":
          \"Der schnelle braune Fuchs\",
        \"alternatives\": [
          \"Der flinke braune Fuchs\"
        ]
      }"
    }
  }]
}




// The LLM responds with
// structured JSON including
// alternative translations

6. Best Practice: Mac M4 + Qwen3-Next

If you have a Mac with Apple Silicon and sufficient RAM, you can achieve excellent translation quality with LM Studio. This configuration shows the optimal setup using a Mac M4 with 128 GB RAM as an example.

Why Qwen3-Next 80B-A3B?

The model qwen3-next-80b-a3b-instruct-mlx is a Mixture-of-Experts model (MoE): 80 billion parameters in total, but only 3 billion active parameters per request. This means:

  • Quality of an 80B model at the speed of a 3B model
  • MLX-optimized -- makes optimal use of Apple Silicon (Neural Engine + GPU)
  • Excellent with multilingual texts and specialized terminology
  • Low latency despite high model quality

Loading the Model

In LM Studio, search for qwen3-next-80b-a3b-instruct-mlx under "Discover" and download it. The model requires approximately 45-50 GB of memory -- with 128 GB RAM there is plenty of room for the browser and other applications.

Recommended Settings

Model qwen3-next-80b-a3b-instruct-mlx
Temperature 0.1 -- low for consistent translations
Max Tokens 2000 -- sufficient for long text blocks
GPU Offload Full (all layers) -- maximum speed

Tips for Optimal Results

  • Use domain contexts -- Select the appropriate context in the Side Panel (Automotive, IT, Medical, Legal). The LLM will deliver significantly more accurate specialized terminology.
  • Enable the cache server -- Previously translated texts are loaded instantly from cache. This saves enormous time, especially on frequently visited sites.
  • Adjust batch size -- On powerful hardware, the batch size can be increased to 30-40 in the settings. This speeds up page translation.
  • Temperature 0.1 -- For translations you want consistency, not creativity. Low temperature = same text yields the same translation.

Performance Comparison

LibreTranslate

  • ~50 texts/second
  • No GPU required
  • General quality
  • No domain context
  • Ideal for quick translations

LM Studio + Qwen3-Next

  • ~10-15 texts/second
  • Apple Silicon GPU
  • High quality + alternatives
  • Domain contexts available
  • Ideal for specialized translations

7. Cache Server

Why Use a Cache?

Without a cache, every text is re-translated on every page load. The cache server stores translations centrally and serves them instantly -- saving time and computing resources.

Text Detected "The quick brown fox"
→
Check Cache Compute SHA-256 hash
→
Hit? Load instantly from cache
→
No Hit Translate via API + cache result

Architecture

The cache uses a dual-hash system:

Domain Hash (Folder)
// All pages of a domain
// share a folder

url_hash = SHA256("svelte.dev")
         = "a1b2c3d4e5f6"

// Result:
// /data/a1b2c3d4e5f6/
Translation Hash (File)
// Each translation is a file

trans_hash = SHA256(
  url + text + "auto:de"
)

// Result:
// /data/a1b2c3d4e5f6/f9e8d7.txt

File Format

Each cache file contains 2 lines (Base64-encoded):

VGhlIHF1aWNrIGJyb3duIGZveA==     # Original
RGVyIHNjaG5lbGxlIGJyYXVuZS4uLg== # Translation

The language pair is encoded in the hash and is not stored separately.

Cache Modes

Server First Server → Local → Translate via API. Default, best performance
Browser Cache First Browser cache → Server → Translate via API. When server is slow
Server Only No browser cache, everything on the server. Saves browser storage
Browser Cache Only Completely offline, no server needed. Data only in browser

8. Setting up the Cache Server

Start Docker

docker run -d \
  --name swt-cache \
  -p 8083:8083 \
  -v swt-cache-data:/data \
  ghcr.io/hallowelt42/swt-cache-server:latest

Settings

Cache Server URL http://localhost:8083
Cache Mode Server Only
Timeout 5000 ms

API Endpoints

Method Endpoint Description
POST /cache/bulk Store translations
POST /cache/get Retrieve translations
GET /cache/url/{hash} Stats per domain
GET /cache/url/{hash}/all All entries for a domain
DEL /cache/url/{hash} Delete domain cache
GET /cache/urls All cached domains
GET /stats Overall statistics
GET /health Health check

Example: Storing Translations

Request
POST /cache/bulk
Content-Type: application/json

{
  "url_hash": "a1b2c3d4e5f6",
  "items": {
    "f9e8d7c6b5a4...": [
      "SGVsbG8gV29ybGQ=",
      "SGFsbG8gV2VsdA=="
    ]
  }
}
Response
HTTP 200 OK

{
  "url_hash": "a1b2c3d4e5f6",
  "created": 1,
  "total": 438
}


// created = new entries
// total   = total count
//           for this domain

9. Frequently Asked Questions

Which backend should I use?

For everyday web pages, LibreTranslate is sufficient -- it is fast and requires few resources. For specialized texts (automotive documentation, medical texts, legal documents), LM Studio is better because it offers domain context prompts and alternative translations.

Do I need the cache server?

No, the cache is optional. Everything works without a cache, but every text is re-translated each time. With a cache, previously translated texts are loaded instantly -- this is significantly faster and saves API calls.

Is my data sent to external servers?

No. Both backends (LibreTranslate and LM Studio) run locally on your machine or within your network. The cache server is self-hosted as well. No data is transmitted to cloud services.

Why does "This Domain" show a count for all pages?

The cache groups translations by domain (e.g. all pages from svelte.dev). This is due to the hash system: the folder name is computed from the hostname. Individual pages are not counted separately.

Can I run LM Studio on a different machine?

Yes. LM Studio starts a local server with an OpenAI-compatible API. All you need is the IP address of the machine and the port (default: 1234). Example: http://192.168.178.45:1234

What happens if the server is unreachable?

Smart Translator shows an error message. The cache server has a timeout (default: 5 seconds) -- if no response is received, the text is translated directly without cache.
