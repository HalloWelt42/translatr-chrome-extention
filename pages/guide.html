1. Was ist Smart Translator?

Smart Translator ist eine Browser-Erweiterung, die Webseiten und markierte Texte direkt im Browser übersetzt. Alle Daten bleiben lokal -- es werden keine externen Cloud-Dienste verwendet.

Ganze Seiten Komplette Webseiten mit einem Klick übersetzen
Textauswahl Markierten Text sofort übersetzen
Side Panel Übersetzungs-Panel neben der Webseite
Cache Übersetzungen zentral speichern und wiederverwenden

2. Übersetzungs-Backends

Smart Translator unterstützt zwei verschiedene Übersetzungs-Backends. Du kannst in den Einstellungen zwischen beiden wechseln.

LibreTranslate

  • Open-Source Übersetzungs-Engine
  • Schnell, keine GPU nötig
  • 30+ Sprachen
  • Automatische Spracherkennung
  • Ideal für allgemeine Texte
  • Selbst gehostet (Docker) oder öffentlich

LM Studio

  • Lokales KI-Modell (LLM)
  • GPU empfohlen (NVIDIA/Apple Silicon)
  • Fachkontexte: Kfz, Medizin, Recht, IT
  • Alternative Übersetzungsvorschläge
  • Ideal für Fachübersetzungen
  • Volle Datenkontrolle, keine Cloud

3. LibreTranslate einrichten

Option A: Docker (empfohlen)

LibreTranslate lässt sich am einfachsten per Docker starten:

docker run -d -p 5000:5000 libretranslate/libretranslate

Nach dem Start ist der Server unter http://localhost:5000 erreichbar.

Option B: Öffentliche Instanz

Du kannst auch eine öffentliche Instanz verwenden. Beachte dabei mögliche Rate-Limits und Datenschutzhinweise. Eine Liste findest du auf der LibreTranslate GitHub-Seite.

Einstellungen

Trage in den Einstellungen die Server-URL ein:

Server-URL http://localhost:5000/translate
API-Key (leer lassen wenn nicht nötig)

So funktioniert die API

Smart Translator sendet Übersetzungsanfragen als JSON an den Server:

Anfrage an den Server
POST /translate
Content-Type: application/json

{
  "q": "The quick brown fox",
  "source": "en",
  "target": "de",
  "format": "text"
}
Antwort vom Server
HTTP 200 OK

{
  "translatedText": "Der schnelle braune Fuchs",
  "detectedLanguage": {
    "confidence": 97.5,
    "language": "en"
  }
}

4. LM Studio einrichten

Schritt 1: LM Studio installieren

Lade LM Studio herunter und installiere es. Lade dann ein Sprachmodell -- empfohlen für Übersetzungen:

  • Gemma 3 12B -- gute Balance aus Qualität und Geschwindigkeit
  • Qwen 2.5 14B -- stark bei mehrsprachigen Texten
  • Llama 3.1 8B -- schnell, für ältere Hardware

Schritt 2: Server starten

Starte in LM Studio den lokalen Server (Tab "Local Server"). Standardmäßig läuft er auf Port 1234.

Schritt 3: Einstellungen

Server-URL http://localhost:1234
Modell (wird automatisch geladen)
Temperatur 0.1 -- niedrig = präzise
Netzwerk-Tipp: Wenn LM Studio auf einem anderen Rechner im Netzwerk läuft, verwende dessen IP-Adresse, z.B. http://192.168.178.45:1234

So funktioniert die API

Smart Translator nutzt die OpenAI-kompatible Chat-API von LM Studio:

Anfrage an LM Studio
POST /v1/chat/completions
Content-Type: application/json

{
  "model": "gemma-3-12b-it",
  "messages": [
    {
      "role": "system",
      "content": "Übersetze EN nach DE.
        Antworte als JSON."
    },
    {
      "role": "user",
      "content": "The quick brown fox"
    }
  ],
  "temperature": 0.1,
  "max_tokens": 2000
}
Antwort von LM Studio
HTTP 200 OK

{
  "choices": [{
    "message": {
      "content": "{
        \"translation\":
          \"Der schnelle braune Fuchs\",
        \"alternatives\": [
          \"Der flinke braune Fuchs\"
        ]
      }"
    }
  }]
}




// Das LLM antwortet mit
// strukturiertem JSON inkl.
// alternativer Übersetzungen

5. Benutzung

Text markieren und übersetzen

Markiere beliebigen Text auf einer Webseite. Es erscheint ein kleines Übersetzungs-Icon. Klicke darauf, um die Auswahl sofort zu übersetzen. Das Ergebnis wird in einem Tooltip angezeigt.

Ganze Seite übersetzen

Klicke auf das Extension-Icon in der Toolbar und dann auf "Seite übersetzen". Alle sichtbaren Texte werden batch-weise übersetzt. Fortschritt und Geschwindigkeit hängen von deinem Backend ab.

Side Panel

Das Side Panel öffnet sich neben der Webseite und bietet:

  • Texteingabe mit Sofort-Übersetzung
  • Seitenübersetzung mit Fortschrittsanzeige
  • Übersetzungsverlauf
  • Cache-Verwaltung mit Statistiken
  • Fachkontext-Auswahl (bei LM Studio)

Popup

Das Popup (Klick auf das Extension-Icon) bietet Schnellzugriff auf:

  • Text eingeben und direkt übersetzen
  • Seite übersetzen / Original wiederherstellen
  • Side Panel öffnen

6. Cache-Server

Warum ein Cache?

Ohne Cache wird jeder Text bei jedem Seitenaufruf neu übersetzt. Der Cache-Server speichert Übersetzungen zentral und stellt sie sofort wieder bereit -- das spart Zeit und Rechenleistung.

Text erkannt "The quick brown fox"
→
Cache prüfen SHA-256 Hash berechnen
→
Treffer? Sofort aus Cache laden
→
Kein Treffer API übersetzen + cachen

Architektur

Der Cache verwendet ein Zwei-Hash-System:

Domain-Hash (Ordner)
// Alle Seiten einer Domain
// teilen einen Ordner

url_hash = SHA256("svelte.dev")
         = "a1b2c3d4e5f6"

// Ergebnis:
// /data/a1b2c3d4e5f6/
Translation-Hash (Datei)
// Jede Übersetzung ist eine Datei

trans_hash = SHA256(
  url + text + "auto:de"
)

// Ergebnis:
// /data/a1b2c3d4e5f6/f9e8d7.txt

Dateiformat

Jede Cache-Datei enthält 3 Zeilen (Base64-kodiert):

VGhlIHF1aWNrIGJyb3duIGZveA==     # Original
RGVyIHNjaG5lbGxlIGJyYXVuZS4uLg== # Übersetzung
YXV0bzpkZQ==                       # Sprachpaar (auto:de)

Cache-Modi

Server zuerst Server → Lokal → API übersetzen. Standard, beste Performance
Lokal zuerst Lokal → Server → API übersetzen. Wenn Server langsam ist
Nur Server Kein LocalStorage, alles auf dem Server. Spart Browser-Speicher
Nur Lokal Komplett offline, kein Server nötig. Daten nur im Browser

7. Cache-Server einrichten

Docker starten

docker run -d \
  --name swt-cache \
  -p 8083:8083 \
  -v swt-cache-data:/data \
  ghcr.io/hallowelt42/swt-cache-server:latest

Einstellungen

Cache Server URL http://localhost:8083
Cache-Modus Nur Server
Timeout 5000 ms

API-Endpunkte

Methode Endpunkt Beschreibung
POST /cache/bulk Übersetzungen speichern
POST /cache/get Übersetzungen abrufen
GET /cache/url/{hash} Stats pro Domain
GET /cache/url/{hash}/all Alle Einträge einer Domain
DEL /cache/url/{hash} Domain-Cache löschen
GET /cache/urls Alle gecachten Domains
GET /stats Gesamtstatistik
GET /health Health-Check

Beispiel: Übersetzungen speichern

Request
POST /cache/bulk
Content-Type: application/json

{
  "url_hash": "a1b2c3d4e5f6",
  "items": {
    "f9e8d7c6b5a4...": [
      "SGVsbG8gV29ybGQ=",
      "SGFsbG8gV2VsdA==",
      "YXV0bzpkZQ=="
    ]
  }
}
Response
HTTP 200 OK

{
  "url_hash": "a1b2c3d4e5f6",
  "created": 1,
  "total": 438
}


// created = neue Einträge
// total   = Gesamtzahl
//           für diese Domain

Verzeichnisstruktur auf dem Server

/data/
  a1b2c3d4e5f6/           # svelte.dev
    f9e8d7c6b5a4...txt    # Übersetzung 1
    1a2b3c4d5e6f...txt    # Übersetzung 2
    ...
  9f8e7d6c5b4a/           # github.com
    abcdef123456...txt
    ...

8. Häufige Fragen

Welches Backend soll ich verwenden?

Für alltägliche Webseiten reicht LibreTranslate -- es ist schnell und braucht wenig Ressourcen. Für Fachtexte (Kfz-Dokumentation, medizinische Texte, juristische Dokumente) ist LM Studio besser, weil es Fachkontext-Prompts und alternative Übersetzungen bietet.

Brauche ich den Cache-Server?

Nein, der Cache ist optional. Ohne Cache funktioniert alles, aber jeder Text wird jedes Mal neu übersetzt. Mit Cache werden bereits übersetzte Texte sofort geladen -- das ist deutlich schneller und spart API-Aufrufe.

Werden meine Daten an externe Server gesendet?

Nein. Beide Backends (LibreTranslate und LM Studio) laufen lokal auf deinem Rechner oder in deinem Netzwerk. Auch der Cache-Server ist self-hosted. Es werden keine Daten an Cloud-Dienste übertragen.

Warum steht bei "Diese Domain" eine Zahl für alle Seiten?

Der Cache gruppiert Übersetzungen nach Domain (z.B. alle Seiten von svelte.dev). Das liegt am Hash-System: der Ordnername wird aus dem Hostnamen berechnet. Einzelne Seiten werden nicht getrennt gezählt.

Kann ich LM Studio auf einem anderen Rechner betreiben?

Ja. LM Studio startet einen lokalen Server mit OpenAI-kompatibler API. Du brauchst nur die IP-Adresse des Rechners und den Port (Standard: 1234). Beispiel: http://192.168.178.45:1234

Was passiert wenn der Server nicht erreichbar ist?

Smart Translator zeigt eine Fehlermeldung. Der Cache-Server hat ein Timeout (Standard: 5 Sekunden) -- wenn keine Antwort kommt, wird der Text ohne Cache direkt übersetzt.
